## Preguntas de reflexion tecnica

1. Que problema concreto resuelve el entorno virtual en un proyecto Python?

El entorno virtual sirve para evitar problemas con las librerías. O sea, que cada proyecto tenga sus propias cosas sin afectar otros proyectos o el sistema.


2. Que diferencia hay entre instalar `Flask` globalmente y hacerlo dentro de `.venv`?

La diferencia es que si instalo Flask global, se instala para todo el computador. Pero si lo hago en .venv, solo funciona dentro de ese proyecto, y es más ordenado y seguro.


3. Por que `requirements.txt` forma parte del proyecto y no de tu maquina personal?

Porque requirements.txt guarda las librerías que usa el proyecto, entonces cualquier persona puede instalar lo mismo y que funcione igual, no depende de mi computador.


4. Cuando ejecutas `python app.py`, que archivo actua como punto de entrada y por que?

El archivo que actúa como punto de entrada es app.py, porque es el que ejecutamos y desde ahí arranca todo el programa.


5. Que relacion hay entre la ruta `/`, la funcion `inicio()` y el archivo `templates/index.html`?

La ruta / llama a la función inicio(), y esa función es la que muestra el archivo templates/index.html. O sea, conecta lo que pasa en el navegador con el archivo HTML.


6. Que evidencia te da la terminal de que el servidor arranco correctamente?

La evidencia es que en la terminal aparece un mensaje como que el servidor está corriendo, por ejemplo “Running on http://…”, eso indica que ya arrancó bien.


7. Si cambias el HTML y el navegador muestra otra cosa, que te demuestra eso sobre el flujo entre backend y frontend en este proyecto?

Eso demuestra que el frontend (HTML) y el backend (Python) están conectados, porque cuando cambio el HTML y recargo, se ve el cambio, entonces sí hay comunicación entre los dos.
